# Лабораторная работа 1: Создание Mark-down сайта

## Цель работы
- Освоить процесс создания статического сайта с использованием генератора документации MkDocs.
- Научиться организовывать структуру документации проекта (портфолио лабораторных работ).
- Изучить базовые принципы работы с системой контроля версий Git и платформой GitHub.
- Развернуть статический сайт с использованием механизма GitHub Pages на домене вида username.github.io.
- Освоить базовую настройку темы оформления и конфигурационного файла mkdocs.yml.

## О лабораторной работе
Данный проект представляет собой статический сайт-портфолио, созданный с помощью генератора документации **MkDocs**. Сайт предназначен для демонстрации лабораторных работ и навыков Python-разработчика. Размещается на **GitHub Pages**.


## Файловое устройство:
- **mkdocs.yml** — главный файл настроек. Здесь указывается название сайта, тема оформления, структура меню и пути к папкам.
- **source/** — папка для исходных Markdown-файлов. Здесь нужно редактировать контент.
- **index.md** — главная страница сайта. Содержит приветствие и краткую информацию.
- **about.md** — страница с информацией об авторе.
- **labs/** — папка с отчетами по лабораторным работам.
- **docs/** — автоматически сгенерированная папка с готовым сайтом. GitHub Pages использует именно её.
- **venv/** — виртуальное окружение Python для изоляции зависимостей.

## Для запуска на своём ПК:
1. Активировать окружение
`venv\Scripts\activate`
2. Запустить сервер
`mkdocs serve`
3. Выполнить сборку сайта
`mkdocs build`
